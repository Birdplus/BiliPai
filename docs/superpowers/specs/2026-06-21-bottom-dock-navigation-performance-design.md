# 底部 Dock 导航性能优化设计

## 目标

- 相邻主页面继续使用现有横向切换动画。
- 跨两页及以上时只呈现起点与目标页，不扫过或组合中间页面。
- 主页面切走后保留 UI 状态，避免返回时重复冷组合、重新加载和丢失列表位置。

## 实现边界

- 仅调整 `MainBottomPagerState`、底栏策略和 `AppNavigation` 的主页面承载方式。
- 沿用现有 `HorizontalPager`、ViewModel 和页面活动态参数，不新增依赖或导航框架。
- 非活动页面保持组合，但继续通过 `isBottomPagerPageActive` 禁止活动页专属加载和高成本效果。
- 播放页创建与释放优化作为后续独立切片处理。

## 切换规则

- 距离为 1：沿用当前 300ms 横向动画。
- 距离大于 1：先静默定位到目标相邻页，再执行一次 300ms 相邻页动画，只显示起点与目标页。
- 动画被新点击取消时，以最新目标为准，最终页必须与 Dock 选中项一致。

## 验证

- 策略单测覆盖相邻和跨页切换计划、页面保留规则。
- 运行导航策略精确单测。
- 运行 `:app:compileDebugKotlin` 验证 Compose 调用。
- 后续设备验证关注：首页到“我的”不再扫过中间页，动态、历史、“我的”返回时保留滚动位置且无重复首载。

## 风险

- 长驻主页面会增加稳定态内存占用；通过活动态参数暂停非活动页加载和视觉效果控制成本。
- Pager 静默定位与动画衔接若方向计算错误会出现闪跳，因此切换计划必须先由纯策略测试约束。
